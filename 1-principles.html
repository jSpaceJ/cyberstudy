Cybersecurity Principles

CIA Triad

The foundation of cybersecurity is the CIA triad:

  • Confidentiality: Ensuring information is not disclosed to unauthorized individuals. - Example: Encrypting a database so only authorized users can read it.
  • Integrity: Ensuring data is accurate and has not been modified. - Example: Digital signatures to prove that a file has not been tampered with.
  • Availability: Ensuring data and services are available when needed. - Example: Redundant servers to prevent downtime from hardware failure.

Design Principles

  • Weakest Link: A system’s security is as strong as its most vulnerable point. Example: Strong encryption is useless if users write passwords on sticky notes.
  • Fail-Safe Defaults: Deny access by default; only grant access when explicitly allowed.
  • Economy of Mechanism: Keep designs simple. Complexity increases chances of errors.
  • Complete Mediation: Every access to every resource should be checked for authorization.
  • Separation of Privilege: Require multiple conditions to grant access (e.g., password + fingerprint).
  • Least Privilege: Give users only the permissions they absolutely need.
  • Psychological Acceptability (Usability): Security must not be too hard to use, or users will bypass it.

Types of Security Goals Beyond CIA

  • Authentication: Proving an entity’s identity (e.g., username + password, biometrics).
  • Non-repudiation: Preventing someone from denying they performed an action (e.g., digital signatures on contracts).
  • Accountability: Keeping logs so actions can be traced to individuals.

Practice Questions

  1. Which principle says security should be simple and easy to use?
    Psychological Acceptability (Usability).
  2. Which part of the CIA triad ensures systems remain usable during a cyberattack?
    Availability.
  3. What principle states that no user should have more privilege than necessary?
    Least Privilege.
